# Wireframes: Transitions & Animations

**Document Version**: 1.0
**Last Updated**: December 2024

---

## Overview

This document defines all transitions, animations, and motion patterns for the Jota mobile app, ensuring consistent and delightful user experience.

---

## 1. Animation Principles

### 1.1 Core Principles

| Principle | Description |
|-----------|-------------|
| **Purposeful** | Every animation serves a functional purpose |
| **Fast** | Animations are quick (200-400ms) and don't impede workflow |
| **Natural** | Motion follows real-world physics (easing, momentum) |
| **Consistent** | Same type of action = same type of animation |
| **Responsive** | Immediate feedback to user input |

### 1.2 Timing Guidelines

| Animation Type | Duration | Easing |
|---------------|----------|--------|
| Micro-interactions | 100-200ms | ease-out |
| Screen transitions | 300-400ms | ease-in-out |
| Complex animations | 400-600ms | spring |
| Loading states | Continuous | linear |

## 10. Performance Guidelines

### 10.1 Animation Performance

| Guideline | Recommendation |
|-----------|----------------|
| Use native driver | Always for transforms and opacity |
| Avoid layout | Don't animate width/height |
| 60fps target | Drop frames = simplify |
| Memory | Dispose animations when off-screen |
| Battery | Reduce animations in low power mode |

### 10.2 Reduced Motion

```
REDUCED MOTION MODE:
- Replace slides with fades
- Remove decorative animations
- Keep functional feedback
- Instant state changes where possible
```

### 10.3 Animation Disable

```
When to disable animations:
- User prefers-reduced-motion: true
- Low battery mode active
- Device is thermal throttling
- Performance drops below 30fps
```
